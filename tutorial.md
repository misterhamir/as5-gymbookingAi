# Complete Step-by-Step Tutorial: FastAPI Booking System + OpenAI Agent

This tutorial will guide you through building a complete Gym Booking API using FastAPI and SQLModel, and then adding an OpenAI-powered Agent that can interact with your database using natural language.

---

## 0. Folder Structure Overview

This tutorial uses the "Monolith" approach where everything runs inside a single FastAPI application. Here is the folder structure you will be building:

```text
as5/
├── alembic/               <-- Generated automatically by Alembic later
├── alembic.ini            <-- Config file generated by Alembic later

    ├── models/            <-- YOU NEED TO CREATE THIS FOLDER
    │   ├── database.py    <-- (Step 3) SQLModel definitions
    │   └── engine.py      <-- (Step 2) Database connection setup
    ├── schemas/           <-- YOU NEED TO CREATE THIS FOLDER
    │   └── bookings.py    <-- (Step 5) Pydantic validation schemas
    ├── router/            <-- YOU NEED TO CREATE THIS FOLDER
    │   ├── bookings.py    <-- (Step 6) API routes for bookings
```

## Part 1: Setting up the FastAPI & Database Core

### Step 1: Install Dependencies
First, make sure you are in your project folder (`as5`). Initialize a virtual environment and install the required packages using `uv`.

```bash
uv add fastapi uvicorn sqlmodel alembic openai pydantic
```

### Step 2: Database Connection (`app/models/engine.py`)
Create a file to manage the SQLite database connection.

```python
# app/models/engine.py
from sqlmodel import create_engine, Session

# This will create a local SQLite file named database.db
engine = create_engine("sqlite:///database.db", echo=True)

def get_db():
    with Session(engine) as session:
        yield session
```

### Step 3: Define Data Models (`app/models/database.py`)
Define the structure of your database tables using SQLModel.

```python
# app/models/database.py
from sqlmodel import SQLModel, Field
from datetime import datetime
from typing import Optional

class User(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    name: str
    email: str

class Schedule(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    class_name: str
    instructor: str
    start_time: datetime
    end_time: datetime

class Booking(SQLModel, table=True):
    id: Optional[int] = Field(default=None, primary_key=True)
    booking_name: str
    payment_status: str = "pending"
    user_id: int = Field(foreign_key="user.id")
    schedule_id: int = Field(foreign_key="schedule.id")
```

### Step 4: Setup Database Migrations with Alembic
Alembic helps manage changes to your database schema.

1. **Initialize Alembic:**
   ```bash
   uv run alembic init alembic
   ```
2. **Configure `alembic.ini`:**
   Open `alembic.ini` and change the `sqlalchemy.url`:
   ```ini
   sqlalchemy.url = sqlite:///database.db
   ```
3. **Configure `alembic/env.py`:**
   Open `alembic/env.py` and modify it to import your models:
   ```python
   # Add these imports at the top:
   from sqlmodel import SQLModel
   from app.models.database import User, Schedule, Booking
   
   # Change target_metadata:
   target_metadata = SQLModel.metadata
   ```
4. **Generate and Run the Migration:**
   ```bash
   uv run alembic revision --autogenerate -m "Initial Gym tables"
   uv run alembic upgrade head
   ```

### Step 5: API Validation Schemas (`app/schemas/bookings.py`)
Pydantic schemas validate the data coming in and out of your API.

```python
# app/schemas/bookings.py
from pydantic import BaseModel
from datetime import datetime

class ScheduleRequest(BaseModel):
    class_name: str
    instructor: str
    start_time: datetime
    end_time: datetime

class BookingRequest(BaseModel):
    booking_name: str
    user_id: int
    schedule_id: int
```

### Step 6: Create API Routes (`app/router/schedule.py` & `bookings.py`)
These files handle the actual API logic (GET, POST).

```python
# app/router/schedule.py
from fastapi import APIRouter, Depends
from sqlmodel import Session, select
from app.models.engine import get_db
from app.models.database import Schedule
from app.schemas.bookings import ScheduleRequest

schedule_router = APIRouter(prefix="/schedule", tags=["Schedule"])

@schedule_router.get("/")
def get_schedules(db: Session = Depends(get_db)):
    return db.exec(select(Schedule)).all()

@schedule_router.post("/")
def create_schedule(schedule_in: ScheduleRequest, db: Session = Depends(get_db)):
    new_schedule = Schedule(**schedule_in.model_dump())
    db.add(new_schedule)
    db.commit()
    db.refresh(new_schedule)
    return new_schedule
```

```python
# app/router/bookings.py
from fastapi import APIRouter, Depends
from sqlmodel import Session, select
from app.models.engine import get_db
from app.models.database import Booking
from app.schemas.bookings import BookingRequest

booking_router = APIRouter(prefix="/bookings", tags=["Bookings"])

@booking_router.get("/")
def get_bookings(db: Session = Depends(get_db)):
    return db.exec(select(Booking)).all()

@booking_router.post("/")
def create_booking(booking_in: BookingRequest, db: Session = Depends(get_db)):
    new_booking = Booking(**booking_in.model_dump())
    db.add(new_booking)
    db.commit()
    db.refresh(new_booking)
    return new_booking
```

### Step 7: The Main Application (`app/main.py`)
Bring it all together.

```python
# app/main.py
from fastapi import FastAPI
from app.router.schedule import schedule_router
from app.router.bookings import booking_router

app = FastAPI(title="Gym Booking API")

app.include_router(schedule_router)
app.include_router(booking_router)
```

**Test it out:** Run `uv run uvicorn app.main:app --reload` and visit `http://127.0.0.1:8000/docs`.

## Part 2: Building the Streamlit AI Agent

Now we add the AI brain using a dedicated Streamlit frontend. This keeps the FastAPI backend focused purely on database logic while Streamlit handles the conversational interface.

### Step 8: Install Streamlit
First, add Streamlit to your project dependencies:
```bash
uv add streamlit openai-agents litellm requests
```

### Step 9: Create the Streamlit App (`streamlit_app.py`)
Create a new file in your project root (not in the `app/` folder) called `streamlit_app.py` and assemble your Agent and User Interface.

```python
# streamlit_app.py
import streamlit as st
import os
import requests
from dotenv import load_dotenv
from openai_agents import Agent, function_tool
from agents.extensions.models.litellm_model import LiteLLMModel

load_dotenv()

# We only need Streamlit for the agent - FastAPI is running on a different port.

# 1. Define Tools as Functions that call your FastAPI Backend
@function_tool
def get_schedules():
    """Retrieve all available gym classes."""
    try:
        response = requests.get("http://127.0.0.1:8000/schedule/")
        response.raise_for_status()
        return response.json()
    except Exception as e:
        return f"Error connecting to server: {str(e)}"

@function_tool
def create_booking(user_id: int, schedule_id: int):
    """Create a new booking for a user in a specific schedule slot."""
    try:
        response = requests.post(
            "http://127.0.0.1:8000/bookings/",
            json={"booking_name": f"Booking_{user_id}_{schedule_id}","user_id": user_id, "schedule_id": schedule_id}
        )
        response.raise_for_status()
        return response.json()
    except Exception as e:
        return f"Error creating booking: {str(e)}"

# 2. Define the Agent
agent = Agent(
    name="Gym Booking Agent",
    instructions="""You are a helpful gym receptionist. 
    Help users view schedules and book classes.
    Always try to answer politely and call the necessary tools when appropriate.
    """,
    model=LiteLLMModel(
        model_name="openai/gpt-4o-mini", # Added openrouter prefix just in case as requested earlier
        api_key=os.getenv("OPENROUTER_API_KEY"),
        base_url="https://openrouter.ai/api/v1",
    ),
    tools=[get_schedules, create_booking],
)

# 3. Streamlit UI
st.title("💪 Gym Booking AI")
st.write("Chat with our virtual receptionist to view classes and book your spot!")

if "messages" not in st.session_state:
    st.session_state.messages = []

# Display chat history
for msg in st.session_state.messages:
    with st.chat_message(msg["role"]):
        st.markdown(msg["content"])

# Handle User Input
if prompt := st.chat_input("Ask about classes or make a booking..."):
    # Render user message
    st.chat_message("user").markdown(prompt)
    st.session_state.messages.append({"role": "user", "content": prompt})
    
    # Render assistant placeholder and run agent
    with st.chat_message("assistant"):
        with st.spinner("Thinking..."):
            try:
                # Run the agent (Syntax might vary slightly based on openai-agents version)
                response = agent.run(message=prompt)
                reply = response.content
            except Exception as e:
                reply = f"Sorry, I encountered an error: {str(e)}"
            
            st.markdown(reply)
            
    # Save assistant message
    st.session_state.messages.append({"role": "assistant", "content": reply})
```

## Part 4: Securing User Passwords

When building the authentication system for your Gym Booking API, you must **never** store plain text passwords in your database. Instead, you need to use a cryptographic hash (like bcrypt) to secure them. Here's a quick guide on how to implement this.

### Step 1: Install Dependencies
First, install the library you need for password hashing, `passlib`, and its optional dependency `bcrypt`:

```bash
uv add "passlib[bcrypt]"
```

### Step 2: Create a Security Utility Module
Create a new file in your project, for example `app/utils/security.py`, to handle the hashing and verification of passwords.

```python
# app/utils/security.py
from passlib.context import CryptContext

# Set up the hashing context using bcrypt
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def get_password_hash(password: str) -> str:
    """Hashes a plain text password."""
    return pwd_context.hash(password)

def verify_password(plain_password: str, hashed_password: str) -> bool:
    """Verifies a plain text password against a hashed one."""
    return pwd_context.verify(plain_password, hashed_password)
```

### Step 3: Update your Pydantic Schemas
In your `app/schemas/gym.py` file, ensure your request schema allows the password field, but your response schema **does not**. This prevents your API from accidentally exposing the hashed password back to the client.

```python
# In app/schemas/gym.py

from typing import List
from pydantic import BaseModel

class UserRequest(BaseModel):
    name: str
    email: str
    password: str

class UserResponse(BaseModel):
    id: int
    name: str
    email: str
    # Notice: No password field here!
```

### Step 4: Hash the Password on User Creation
Modify your user creation route (e.g., in `app/router/login.py`) to hash the password before saving the model to the database.

```python
# In app/router/login.py

from fastapi import APIRouter, Depends
from sqlmodel import Session
from app.models.engine import get_db
from app.models.database import User
from app.schemas.gym import UserRequest, UserResponse
from app.utils.security import get_password_hash # Import your new utility!
from typing import List

login_router = APIRouter(prefix="/login", tags=["login"])

@login_router.post("/", response_model=UserResponse)
def create_user(user: UserRequest, db: Session = Depends(get_db)):
    # 1. Hash the password from the request
    hashed_password = get_password_hash(user.password)
    
    # 2. Exclude the plain password when creating the dictionary of values
    user_data = user.dict(exclude={"password"})
    
    # 3. Create the Database User model, passing the hashed password instead
    new_user = User(**user_data, password=hashed_password)
    
    db.add(new_user)
    db.commit()
    db.refresh(new_user)
    
    # 4. Return the new user. FastAPI uses `response_model=UserResponse` 
    # to automatically strip the password field out of the JSON response.
    return new_user

@login_router.get("/", response_model=List[UserResponse])
def get_users(db: Session = Depends(get_db)):
    return db.query(User).all()
```
